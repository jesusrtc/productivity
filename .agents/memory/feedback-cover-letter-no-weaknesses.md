In the `projects/resume/CV` project, **never put weaknesses, gaps, or "I don't have X / I would lean on the team for X" framing into `cover-letter.tex`.** Honest gaps go in `fit.md` (internal planning) only. The cover letter is an outbound sales doc to the hiring manager and should close on strengths and intent, not caveats.

**Why:** Jesus flagged this after the 2026-05-30 Snowflake application, where para 4 named "source code reviews across Java and Go, and cloud security tooling depth" as gaps and said "I would lean on the team's specialists." That kind of caveat-on-the-close hurts the application — gaps come up naturally in screening conversations and don't need to be pre-volunteered in writing.

**How to apply:**
- Para 4 (closing) of every cover letter must close on strengths + intent. Common shape: "I would bring <strengths>... and would welcome the opportunity to discuss..."
- Never name a missing skill, missing years, missing certification, or absence of experience anywhere in the cover letter (paras 1–4 inclusive).
- Keep the gap analysis in `fit.md` under "Gaps and honest notes" — that's what it's for.
- This rule is also written into `projects/resume/CV/CLAUDE.md` under "When Creating a New Application", so it loads automatically inside that project; this memory is the cross-session backup.
